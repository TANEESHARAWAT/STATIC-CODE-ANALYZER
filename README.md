# Static Code Analyzer

A static code analyzer for **Python** and **Java** built with [ANTLR4](https://www.antlr.org/).
Parses source files, walks the AST, and reports rule violations with line numbers and severity levels.

## Features
- Supports Python 3 and Java 8
- Rule-based analysis using the ANTLR4 Visitor pattern
- Severity levels: `ERROR` / `WARNING` / `INFO`
- Output formats: plain text, JSON
- Clean CLI interface

## Setup

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Download grammar files
Get them from https://github.com/antlr/grammars-v4 and place in `grammars/`:
- `grammars/Python3.g4`
- `grammars/Java8.g4`

### 3. Generate lexer/parser
```bash
antlr4 -Dlanguage=Python3 -visitor -o generated/python3 grammars/Python3.g4
antlr4 -Dlanguage=Python3 -visitor -o generated/java8  grammars/Java8.g4
```

## Usage
```bash
python main.py samples/bad_code.py
python main.py samples/BadCode.java
python main.py samples/bad_code.py --format json
```

## Rules
See [docs/rules.md](docs/rules.md) for the full reference.

| ID    | Language | Severity | Description                       |
|-------|----------|----------|-----------------------------------|
| PY001 | Python   | WARNING  | Function name not snake_case      |
| PY002 | Python   | WARNING  | Too many parameters (> 5)         |
| PY003 | Python   | ERROR    | Bare except clause                |
| PY004 | Python   | WARNING  | Ambiguous variable name           |
| JV001 | Java     | ERROR    | Class name not PascalCase         |
| JV002 | Java     | WARNING  | Method name not camelCase         |
| JV003 | Java     | ERROR    | Empty catch block                 |
| JV005 | Java     | INFO     | System.out.println left in code   |
